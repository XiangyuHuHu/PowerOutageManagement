### 测试类型覆盖

#### 1. 单元测试 (Unit Tests)
- **认证模块**: 用户注册、登录、权限验证、密码安全
- **应用逻辑**: Flask路由、中间件、错误处理
- **MQTT通信**: 消息发布、订阅、处理
- **设备管理**: 状态监控、告警处理

#### 2. 集成测试 (Integration Tests)
- **端到端工作流**: 完整的停电/送电申请流程
- **角色权限集成**: 多角色协作测试
- **系统集成**: 用户-申请-设备关联测试

#### 3. API测试 (API Tests)
- **RESTful接口**: 所有主要API端点
- **数据验证**: 请求/响应格式验证
- **错误处理**: 异常情况处理测试

#### 4. 安全测试 (Security Tests)
- **SQL注入防护**: 恶意SQL输入测试
- **XSS防护**: 跨站脚本攻击防护
- **认证授权**: 未授权访问测试

#### 5. 性能测试 (Performance Tests)
- **并发操作**: 多用户并发请求
- **系统稳定性**: 连续请求压力测试
- **响应时间**: 请求处理时间监控

## 🔧 技术栈

### 测试框架
- **pytest**: 主测试框架
- **pytest-flask**: Flask应用测试插件
- **pytest-cov**: 代码覆盖率分析
- **pytest-mock**: Mock对象支持

### 测试工具
- **unittest.mock**: Python内置Mock库
- **factory-boy**: 测试数据工厂
- **faker**: 假数据生成
- **httpx**: HTTP客户端测试
- **responses**: HTTP响应Mock

## 🚫 已知限制

### 数据库依赖
- 大多数失败测试都是由于缺少MySQL数据库连接
- 返回401未授权错误，因为登录验证需要数据库支持

### 环境依赖
- 测试环境与生产环境存在差异
- 某些功能需要完整的基础设施支持

### 建议改进
1. **Mock数据库**: 使用SQLite内存数据库进行测试
2. **测试数据**: 创建标准化测试数据集
3. **CI/CD集成**: 自动化测试流水线
4. **覆盖率提升**: 目标达到80%以上覆盖率

## 📋 使用指南

### 运行所有测试
```bash
pytest tests/
```

### 运行特定模块测试
```bash
pytest tests/test_auth.py -v
```

### 生成覆盖率报告
```bash
pytest tests/ --cov=. --cov-report=html
```

### 运行特定类型测试
```bash
pytest tests/ -m "unit"      # 单元测试
pytest tests/ -m "integration" # 集成测试
```
