# HBuilderX 启动指南

## 🚀 启动步骤

### 1. 打开项目
1. 打开 HBuilderX
2. 点击 `文件` → `打开目录`
3. 选择 `miniprogram` 文件夹
4. 等待项目加载完成

### 2. 检查项目结构
确保以下文件存在：
- ✅ `pages.json` - 页面配置
- ✅ `manifest.json` - 应用配置
- ✅ `App.vue` - 应用入口
- ✅ `main.js` - 主入口
- ✅ `pages/` - 页面文件夹

### 3. 配置启动平台

#### 方式一：H5浏览器（推荐）
1. 在HBuilderX中，点击工具栏的 `运行` 按钮
2. 选择 `运行到浏览器` → `Chrome`
3. 等待编译完成，会自动打开浏览器

#### 方式二：微信开发者工具
1. 下载并安装微信开发者工具
2. 在HBuilderX中，点击 `运行` → `运行到小程序模拟器` → `微信开发者工具`
3. 首次运行需要配置微信开发者工具路径

#### 方式三：手机预览
1. 在HBuilderX中，点击 `运行` → `运行到手机或模拟器` → `运行到Android App基座`
2. 需要先安装HBuilderX基座应用

### 4. 检查新页面是否生效

#### 测试注册功能
1. 启动项目后，访问登录页面
2. 点击 `注册新用户` 按钮
3. 应该跳转到注册页面

#### 测试管理员功能
1. 使用管理员账号登录（admin/admin）
2. 进入管理员页面
3. 点击各个功能卡片，测试跳转

## 🔧 常见问题解决

### 问题1：新页面无法访问
**解决方案：**
1. 检查 `pages.json` 是否正确配置
2. 重新编译项目：`运行` → `重新编译`
3. 清除缓存：`运行` → `清除缓存并运行`

### 问题2：页面跳转失败
**解决方案：**
1. 检查页面路径是否正确
2. 确认页面文件是否存在
3. 检查控制台错误信息

### 问题3：样式不生效
**解决方案：**
1. 检查CSS语法是否正确
2. 重新编译项目
3. 清除浏览器缓存

### 问题4：API请求失败
**解决方案：**
1. 确保Web后端已启动（`python server.py`）
2. 检查API地址是否正确（`http://localhost:5050`）
3. 检查网络连接

## 📋 项目配置检查

### 1. 检查pages.json
确保包含所有新页面：
```json
{
  "pages": [
    {"path": "pages/login/login", "style": {"navigationBarTitleText": "用户登录"}},
    {"path": "pages/register/register", "style": {"navigationBarTitleText": "用户注册"}},
    {"path": "pages/user_management/user_management", "style": {"navigationBarTitleText": "用户管理"}},
    {"path": "pages/stats/stats", "style": {"navigationBarTitleText": "系统统计"}},
    {"path": "pages/repair/repair", "style": {"navigationBarTitleText": "检修管理"}},
    // ... 其他页面
  ]
}
```

### 2. 检查manifest.json
确保H5配置正确：
```json
{
  "h5": {
    "router": {
      "mode": "history",
      "base": ""
    }
  }
}
```

## 🚨 故障排除

### 如果仍然看不到新页面：
1. **强制重新编译**
   - 点击 `运行` → `重新编译`
   - 等待编译完成

2. **清除缓存**
   - 点击 `运行` → `清除缓存并运行`
   - 重新启动项目

3. **检查文件路径**
   - 确认新页面文件在正确位置
   - 检查文件名是否正确

4. **重启HBuilderX**
   - 完全关闭HBuilderX
   - 重新打开项目

### 如果API请求失败：
1. **检查后端服务**
   ```bash
   cd web-backend
   python server.py
   ```

2. **检查网络连接**
   - 确保可以访问 `http://localhost:5050`
   - 检查防火墙设置

3. **检查API地址**
   - 确认代码中的API地址正确
   - 测试API是否响应

---

**提示：** 如果按照以上步骤仍然无法看到新页面，请尝试在HBuilderX中右键点击项目，选择"重新识别项目类型"。
